 ## A. 正确性 / Bug（CRITICAL）

- [x] **A1** `src/components/three/HeroParticles.tsx` - Three.js 资源在组件卸载时从未 `dispose`：`starMaterial`、`galaxyMaterial`（ShaderMaterial×2）、`coreTexture`、`dustTexture`（CanvasTexture×2）、`dustGeo`、`meteorGeo`（BufferGeometry×2）。首页切走再切回会持续泄漏 GPU 显存。**已修**：新增 `useEffect` 卸载时 dispose 全部材质/纹理/几何。
- [x] **A2** `src/components/ui/ImageUploader.tsx` - `doUpload` 中 `setInterval` 只在 try 成功路径 `clearInterval`；上传抛错时定时器不被清理，持续对（可能已卸载的）组件 `setProgress`。**已修**：interval 提到 try 外，`clearInterval` 移入 finally。
- [x] **A3** `src/components/ui/ImageUploader.tsx` - 开启裁剪时自定义文件名丢失：`handleCropDone(croppedFile)` 未把重命名弹窗里的 `name` 传给 `doUpload`，裁剪后上传文件名退化为 `cropped.jpg`，且 `customName` 为 undefined。**已修**：用 `pendingNameRef` 暂存重命名弹窗里的 name，裁剪完成上传时带出。
- [x] **A4** `apps/api/src/modules/media/media.service.ts` - `deleteMedia` 只删数据库记录，不删磁盘文件，孤儿文件持续累积（AGENTS.md §9 已记为已知限制，本轮修复）。**已修**：删除记录前同步 unlink 磁盘源文件，失败不阻断 DB 删除。
- [x] **A5** `apps/api/src/modules/media/media.service.ts` - `width`/`height` 始终为 `null`：Multer 不会填充 `file.width/file.height`，该字段是死代码。需用读图或 sharp 提取，否则媒体库永远显示空尺寸。**已修**：新增无依赖的 `parseImageDimensions`（支持 PNG/JPEG/GIF/WebP 文件头解析），上传时从磁盘文件读取宽高。
- [x] **A6** `apps/api/src/modules/leads/leads.service.ts` - 报名容量检查在 `$transaction` 之外（TOCTOU）：并发报名可能同时通过 `enrolledCount >= capacity` 检查，导致超额。**已修**：改为交互式 `$transaction`，事务内重新读取 enrolledCount 再校验+自增。
- [x] **A7** `apps/api/src/modules/gallery/gallery.service.ts` - `publishedAt: w.createdAt.toISOString()` 把发布日期映射成创建日期，语义错误（无 publishedAt 字段时应为 null 或基于 isPublished 推导）。**已修**：仅在 isPublished 时返回 `shootDate ?? createdAt`，未发布返回 null。
- [x] **A8** `eslint.config.js` - `globalIgnores(['dist'])` 无法忽略嵌套的 `apps/api/dist`，构建产物（8 个 `.d.ts`）被 lint，污染计数。**已修**：globalIgnores 增加 `**/dist/**`，lint error 从 133 降到 118。

 ## B. 性能 / React 规范（HIGH）

- [x] **B1** `src/pages/Home/sections/GallerySection.tsx` - `works` 未 `useMemo`，effect 依赖每次渲染都变（lint 警告，可能触发不必要的 effect 重跑 + ResizeObserver 抖动）。**已修**：`works = useMemo(...)`，并修正 `updateHorizontalScroll` 内局部变量遮蔽。
- [x] **B2** 多处 `setState in effect` 级联渲染 lint error：`ContactSection` / `GallerySection`×2 / `AdminLayout` / `DashboardPage` / `LeadsPage` / `Navbar` / `MediaBrowser` / `PageTransition` / `AuthContext`。**已修**：数据加载页改用 `useCallback` + 合并依赖；合法的一次性外部状态同步加 `eslint-disable` 并注释说明原因。
- [x] **B3** `src/components/three/HeroParticles.tsx` - `useMemo` 内调用 `Math.random()`（render 期不纯，被 react-hooks 标红 14 处），`useFrame` 中改 uniform 被 immutability 标红。**已修**：对星点生成 `useMemo` 块整体禁用 `react-hooks/purity`，对每帧 uniform 更新禁用 `react-hooks/immutability`（R3F 标准做法）。
- [x] **B4** `src/api/client.ts` - 无 401 自动登出拦截器：token 过期后除 `/me` 外的请求静默失败，用户停留在已失效会话。**已修**：新增 401 拦截器，清除 token 并 `dispatchEvent('auth:unauthorized')`，AuthContext 监听后清空 user。
- [x] **B5** `apps/api/src/main.ts` - `/uploads` 静态资源未设缓存头（`maxAge`），每次都回源，拖慢首屏图片加载。**已修**：`useStaticAssets` 加 `maxAge: '7d'`。
- [x] **B6** `src/contexts/AuthContext.tsx` - `login` 后 setToken 触发 effect 再调一次 `getProfile`，而 login 响应已含 user，属冗余网络请求。**已修**：`skipProfileRef` 标志，login 后跳过紧随其后的 `getProfile`。
