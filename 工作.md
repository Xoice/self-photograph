## 项目概况

**技术栈：**
- 前端：Vite (rolldown-vite) + React 19 + TypeScript + MUI 7 + GSAP + Lenis + React Three Fiber
- 数据层：Axios + TanStack Query + React Hook Form + Zod
- 后端：NestJS + TypeScript + Prisma + SQLite (本地) / PostgreSQL (生产) + JWT + bcryptjs + Multer
- 数据库：本地用 SQLite，部署用 PostgreSQL

**项目定位：** 个人摄影展示站，前后端分离，单 admin 用户

## 注意事项

1. **GSAP 动画** — 所有 Section 的 useGSAP 动画必须保留，改造时不能破坏
2. **Gallery 横向滚动** — 最高风险点，useGSAP 需要 `dependencies: [works.length]`
3. **Lenis + ScrollTrigger** — 异步数据加载后必须 `ScrollTrigger.refresh()`
4. **Mock 模式** — `VITE_USE_MOCK=true` 时用 mock adapter，`false` 时用真实 API
5. **响应拦截器** — 已解包为 `body.data`，API 函数直接返回领域对象
6. **SQLite 模式** — 本地开发用 SQLite，部署时切换 PostgreSQL
7. **Multer 文件** — 手动设置 `Content-Type: multipart/form-data` 会导致上传失败，让 Axios 自动处理
8. **DTO 校验** — Admin 接口已使用 DTO，不要回退到 `any`
 9. **SQLite 限制** — 不支持 `mode: insensitive`，查询时需注意
