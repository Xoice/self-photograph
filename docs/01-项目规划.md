# 项目规划

## 1. 当前定位

`self_photograph` 当前已经是一个完整的前后端分离项目，不再是早期的静态前端站点。

项目定位：

- 摄影作品展示站
- 研学活动展示与报名入口
- 内容可通过后台维护
- 单管理员模型
- 强视觉与动效优先

## 2. 当前架构判断

### 仓库形态

当前仓库已经是单仓全栈结构：

```text
self_photograph/
├─ src/          # 前端站点 + 后台前端
├─ apps/api/     # NestJS 后端
├─ docs/         # 文档体系
└─ 工作.md        # 交接与推进摘要
```

### 技术架构

- 前端：`Vite (rolldown-vite) + React 19 + TypeScript + MUI 7 + GSAP + Lenis + R3F`
- 前端数据层：`Axios + TanStack Query + React Hook Form + Zod`
- 后端：`NestJS + Prisma + JWT + bcryptjs + Multer`
- 数据库：开发使用 `SQLite`
- 生产目标：`PostgreSQL + 云端媒体存储`

## 3. 业务目标

当前项目的长期目标不是只做展示，而是形成一套可持续维护的摄影内容平台。

### 对外能力

- 展示个人品牌与摄影风格
- 展示精选作品与作品详情
- 展示视频内容
- 展示研学活动详情
- 收集联系信息和报名线索

### 对内能力

- 管理员登录
- 内容维护（作品/视频/研学 CRUD）
- 封面图上传与复用（ImageUploader + ImageCropper + MediaBrowser）
- 站点配置统一维护
- 媒体库管理

## 7. 当前关键风险

### 动画与异步数据耦合

- `GallerySection` 使用 `ScrollTrigger` 横向滚动
- 异步数据变化可能影响滚动宽度与触发位置
- 数据变化后必须关注 `ScrollTrigger.refresh()`

### Lenis 与锚点路由

- 当前站点大量使用首页 hash 定位
- 动态渲染后若元素尚未挂载，滚动会失准
- `ScrollToTop` 已做补偿，改动时要保持该机制

### 后台仍是 MVP

- 已能用，但表单字段不完整
- 线索管理页面缺失
- 分类、标签、排序管理仍可继续增强

### 媒体存储尚未生产化

- 当前 `/uploads` 适合本地开发
- 上线前建议替换为 `Cloudinary / S3 / OSS / COS`

## 8. 当前路线图

### 阶段 A：内容管理增强

- 为作品增加分类、标签、更多描述字段
- 为研学增加行程、费用项的后台编辑
- 为视频增加平台、时长、分类校验

### 阶段 B：线索与运营增强

- 增加后台线索管理页
- 支持线索状态变更
- 增加报名导出或筛选

### 阶段 C：部署与生产化

- 切换 PostgreSQL
- 切换云存储
- 配置域名、HTTPS、反向代理
- 做备份与日志

### 阶段 D：质量保障

- 增加单元测试与 E2E
- 增加 CI/CD
- 增加错误监控与性能分析

## 9. 文档与代码的关系

本项目文档应服务于"继续构建当前已落地系统"，因此：

- 文档必须以现有代码为准
- 规划文档不能再把已完成模块写成"待建设"
- 所有后续规划应从当前能力出发递进扩展

## 10. 建议的下一步

如果继续推进项目，推荐顺序如下：

1. 先完善后台内容编辑能力
2. 再补线索管理页
3. 再做媒体云存储迁移
4. 然后做部署与 SEO
5. 最后补测试与 CI/CD
