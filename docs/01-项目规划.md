# 项目规划

## 1. 当前定位

`self_photograph` 当前已经是一个完整的前后端分离项目，不再是早期的静态前端站点。

项目定位：

- 摄影作品展示站
- 研学活动展示与报名入口
- 内容可通过后台维护
- 单管理员模型
- 强视觉与动效优先

## 2. 当前架构判断

### 仓库形态

当前仓库已经是单仓全栈结构：

```text
self_photograph/
├─ src/          # 前端站点 + 后台前端
├─ apps/api/     # NestJS 后端
├─ docs/         # 文档体系
└─ 工作.md        # 交接与推进摘要
```

### 技术架构

- 前端：`Vite (rolldown-vite) + React 19 + TypeScript + MUI 7 + GSAP + Lenis + R3F`
- 前端数据层：`Axios + TanStack Query + React Hook Form + Zod`
- 后端：`NestJS + Prisma + JWT + bcryptjs + Multer`
- 数据库：开发使用 `SQLite`
- 生产目标：`PostgreSQL + 云端媒体存储`

## 3. 业务目标

### 对外能力

- 展示视频内容

- 封面图上传与复用（ImageUploader + ImageCropper + MediaBrowser）

## 7. 当前关键风险

### 动画与异步数据耦合

- `GallerySection` 使用 `ScrollTrigger` 横向滚动
- 异步数据变化可能影响滚动宽度与触发位置
- 数据变化后必须关注 `ScrollTrigger.refresh()`

### Lenis 与锚点路由

- 当前站点大量使用首页 hash 定位
- 动态渲染后若元素尚未挂载，滚动会失准
- `ScrollToTop` 已做补偿，改动时要保持该机制

### 媒体存储尚未生产化

- 当前 `/uploads` 适合本地开发
- 上线前建议替换为 `Cloudinary / S3 / OSS / COS`

### ThrottlerModule 内存存储限制

- 当前 `ThrottlerModule` 使用内存存储
- 多实例部署时限流不共享，应在反向代理层（Nginx）做限流

### 阶段 A：部署与生产化

**目标**：让项目可以安全部署到生产环境。

#### A1. 数据库迁移

- Prisma schema 切换 `provider` 为 `postgresql`
- 更新 `DATABASE_URL` 连接字符串
- 验证迁移兼容性

#### A2. 媒体存储迁移

- 接入 Cloudinary / S3 / OSS
- 保留本地上传作为 fallback
- 媒体删除同步清理云端文件

#### A3. 部署配置

- 前端构建 + Nginx 反向代理
- 后端 PM2 / Docker 部署
- HTTPS 配置
- 环境变量管理
- 做备份与日志

#### A4. SEO 增强

- 动态 meta 标签（react-helmet 或手动注入）
- sitemap.xml 生成
- robots.txt
- 结构化数据（JSON-LD）

### 阶段 B：质量保障

**目标**：建立测试和自动化体系，提升用户体验。

#### B1. 性能优化

#### B2. 测试框架

#### B3. CI/CD

#### B4. 监控

- 错误监控
- 性能分析
