# 后端文档

## 1. 后端定位

后端当前已经是项目的正式组成部分，而不是规划中的未来模块。

后端职责包括：

- 输出公开站点数据
- 提供后台管理接口
- 管理认证与权限
- 保存联系/报名线索
- 提供媒体上传与媒体列表

## 2. 当前技术栈

- `NestJS 10`
- `TypeScript`
- `Prisma 6`
- `JWT` + `passport-jwt`
- `bcryptjs`
- `Multer`
- `class-validator` + `class-transformer`

数据库现状：

- 本地开发使用 `SQLite`
- 生产建议迁移到 `PostgreSQL`

## 4. 模块划分

### `auth`

负责：

- 管理员登录（`POST /admin/auth/login`）
- 当前用户信息获取（`GET /admin/auth/me`）
- JWT 策略校验（`jwt.strategy.ts` 在校验时不仅解析 token，还会查询数据库确认用户存在且 `status` 为 `active`，否则拒绝认证）

### `site`

负责：

- 公开站点配置查询（`GET /site/config`）
- 后台站点配置获取（`GET /admin/site/config`）
- 后台站点配置修改（`PUT /admin/site/config`，含 DTO 校验）

### `gallery`

负责：

- 公开分类查询（`GET /gallery/categories`）
- 公开作品列表（`GET /gallery/works`）
- 公开作品详情（`GET /gallery/works/:slug`）
- 后台作品 CRUD（含 DTO 校验）
- 后台分类 CRUD：
  - `GET /admin/gallery/categories`
  - `POST /admin/gallery/categories`
  - `PATCH /admin/gallery/categories/:id`
  - `DELETE /admin/gallery/categories/:id`

### `videos`

负责：

- 公开视频列表（`GET /videos`）
- 后台视频 CRUD（含 DTO 校验）

### `workshops`

负责：

- 公开活动列表（`GET /workshops`）
- 公开活动详情（`GET /workshops/:slug`）
- 后台活动 CRUD（含 DTO 校验）

### `leads`

负责：

- 联系表单提交（`POST /leads/contact`）
- 活动报名提交（`POST /leads/workshop-enrollments`，含容量检查）
- 报名重复检查：同一 `workshopId` + `phone` 组合不允许重复报名
- 后台线索查询与状态修改

### `media`

负责：

- 图片上传（`POST /admin/media/upload`，上传时通过 partial read 解析图片宽高）
- 媒体列表（`GET /admin/media`）
- 媒体重命名（`PATCH /admin/media/:id`）
- 媒体删除（`DELETE /admin/media/:id`，删除时同时删除磁盘文件）

### `health`

负责：

- 服务健康检查（`GET /health`）

## 5. 应用级配置

### `main.ts` 职责

`apps/api/src/main.ts` 当前只负责基础设施层配置：

- 设置全局前缀 `/api/v1`
- 排除 `health`
- 开启 CORS（`origin` 来自环境变量）
- 挂载 Helmet 安全头
- 开放静态目录 `/uploads`
- 调用 `enableShutdownHooks()`

### `app.module.ts` DI 注册

`ValidationPipe`、`ResponseTransformer`、`ApiExceptionFilter` 不再在 `main.ts` 中通过 `new` 实例化挂载，而是通过依赖注入在 `app.module.ts` 中注册：

- `APP_PIPE` → `ValidationPipe`（whitelist + forbidNonWhitelisted + transform）
- `APP_INTERCEPTOR` → `ResponseTransformer`（统一响应格式）
- `APP_FILTER` → `ApiExceptionFilter`（统一错误格式）
- `APP_GUARD` → `ThrottlerGuard`（限流）

这样所有全局组件都走 NestJS 的 DI 容器，便于测试与替换。

## 6. 当前接口风格

### 全局前缀

- 公开接口：`/api/v1/*`
- 后台接口：`/api/v1/admin/*`
- 健康检查：`/health`

### 响应格式

统一输出：

```json
{
  "code": 0,
  "message": "ok",
  "data": {}
}
```

### 认证方式

- 后台接口统一使用 `Bearer Token`
- token 来源于登录接口返回的 `accessToken`
- `JwtAuthGuard` 保护后台路由
- `CurrentUser` 装饰器提取当前用户信息

## 8. 当前数据模型说明

### 用户

用于后台登录。

关键字段：

- `email`（唯一）
- `passwordHash`
- `name`
- `role`（默认 `admin`）
- `status`（默认 `active`）

### 站点配置

控制首页及联系信息。

关键字段：

- `brandName`
- `heroTitle`
- `heroSubtitle`
- `bioTitle`
- `bioContent`
- `contactPhone`
- `contactEmail`
- `contactWechat`
- `locationText`
- `bilibiliUrl`（可选）
- `footerText`

### 画廊分类

支持树形结构：

- `name`
- `slug`（唯一）
- `parentId`（可选，自关联）
- `sortOrder`
- `isVisible`

### 画廊标签

- `name`
- `slug`（唯一）
- 与 `GalleryWork` 多对多

### 作品

当前支持：

- 分类关联（`categoryId` 可选）
- 多标签
- 封面图 + 缩略图
- 拍摄信息（日期/地点/相机）
- 发布状态
- 精选标记
- 排序

### 视频

当前支持：

- 标题、slug、描述
- 平台与视频链接
- 封面图
- 时长（秒）
- 分类
- 发布状态
- 排序

### 研学

当前支持：

- 基础活动信息
- 亮点（`WorkshopHighlight`，级联删除）
- 行程（`WorkshopItinerary`，级联删除）
- 费用项（`WorkshopFeeItem`，级联删除）
- 名额与报名数
- 状态控制（draft / published / registration_open / full / closed）
- 价格文本与分

### 线索

分为两类：

- `ContactLead`：联系表单
- `WorkshopEnrollment`：研学报名（关联 Workshop）

### 媒体

当前保存：

- 文件名
- 文件类型
- MIME
- URL
- 宽高
- 大小
- 上传人

## 9. DTO 校验

所有模块的 Admin 控制器已全部接入 DTO，不再使用 `any`。配合全局 `ValidationPipe`（`whitelist` + `forbidNonWhitelisted: true` + `transform`），非法字段会被自动拦截并返回 400 错误。

已实现的 DTO：

- `gallery/dto/gallery.dto.ts`：
  - `CreateWorkDto`, `UpdateWorkDto`
  - `CreateCategoryDto`, `UpdateCategoryDto`
- `videos/dto/videos.dto.ts`：`CreateVideoDto`, `UpdateVideoDto`
- `workshops/dto/workshops.dto.ts`：`CreateWorkshopDto`, `UpdateWorkshopDto`
- `leads/dto/leads.dto.ts`：
  - `CreateContactLeadDto`
  - `CreateEnrollmentDto`
  - `UpdateLeadStatusDto`
- `site/dto/site.dto.ts`：`UpdateSiteConfigDto`

`forbidNonWhitelisted: true` 确保请求体中出现 DTO 未定义的字段时直接拒绝，防止参数注入。

- 生产环境尚未完成 PostgreSQL 与云存储切换
