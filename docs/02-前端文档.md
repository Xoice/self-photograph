# 前端文档

## 1. 前端定位

前端当前同时承担两个角色：

- 公开站点
- 管理后台前端

两者共用同一个 React 应用，通过路由拆分。

## 3. 路由结构

当前 `src/App.tsx` 中已定义的主要路由：

### 公开站点

- `/` - 首页（Hero + Bio + Gallery + Video + PhotographyStudy + Contact）
- `/gallery` - 画廊列表页
- `/gallery/:slug` - 作品详情页
- `/workshops` - 研学列表页
- `/workshops/:slug` - 研学详情页
- `/photographystudy/kenya-expedition` - 旧页面兼容
- `*` - 404 页面

### 后台

- `/admin/login` - 登录页
- `/admin` - 后台首页（概览）
- `/admin/works` - 作品管理
- `/admin/videos` - 视频管理
- `/admin/workshops` - 研学管理
- `/admin/workshops/new` - 研学新建
- `/admin/workshops/edit/:id` - 研学编辑
- `/admin/categories` - 分类管理
- `/admin/leads` - 线索管理
- `/admin/site-config` - 站点配置
- `/admin/media` - 媒体库

所有后台子路由使用 `AdminLayout` 嵌套，非详情页使用 `lazy()` + `Suspense` 懒加载。

## 5. API 客户端机制

### 响应拦截器

`src/api/client.ts` 中响应拦截器已解包统一格式：

- 成功：返回 `body.data`（直接是领域对象，不是 AxiosResponse）
- 失败：抛出 `ApiError`（含 code + message）

### 请求拦截器

- 自动从 `localStorage` 读取 `auth_token`
- 附加 `Authorization: Bearer <token>` 头

### Mock 切换

- `VITE_USE_MOCK=true`：`apiClient` 使用 `createMockAdapter()` 替代真实请求
- `VITE_USE_MOCK=false`：走真实后端 API
- Mock 数据在 `src/mocks/` 中，按 URL 路径匹配返回

## 6. 公开站点模块

### `HeroSection`

- 使用站点配置接口渲染主标题与副标题
- 保留 Three.js 粒子背景
- 保留首屏入场动画

### `BioSection`

- 从站点配置中读取简介内容
- 保留滚动入场动画

### `GallerySection`

- 首页精选画廊
- 数据来源：`useGalleryWorks({ featured: true, pageSize: 8 })`
- 使用 `GSAP + ScrollTrigger` 做横向滚动
- `useGSAP` 添加 `dependencies: [works.length]` 确保数据加载后重跑动画
- 点击卡片跳转到 `/gallery`
- "查看全部作品"按钮

### `GalleryListPage`

- 展示全部作品（分页）
- 分类筛选接通 API
- GSAP 入场动画
- 点击卡片跳转 `/gallery/:slug`

### `WorkDetailPage`

- 根据 `slug` 拉取单个作品详情
- 展示分类、标签、时间、地点、相机信息、更多图片
- 加载态 / 错误态 / 空态
- `Lightbox` 图片全屏查看
- `ShareButton` 分享

### `VideoSection`

- 数据来源：`useVideos`
- 支持视频封面、分类、时长与跳转链接

### `PhotographyStudy`

- 数据来源：`useWorkshops`
- 首页展示研学卡片
- 状态芯片颜色：`registration_open` 绿 / `full` 红 / 其他灰
- 跳转到 `/workshops/:slug`

### `WorkshopDetailPage`

- 动态详情页
- 渲染活动介绍、亮点、行程、费用、联系方式
- 联系方式从 `useSiteConfig` 获取
- 报名表单使用 `useSubmitEnrollment`，复用 `EnrollmentDialog`

### `ContactSection`

- 联系方式来自 `useSiteConfig`
- 表单使用 `React Hook Form + Zod`
- 提交走 `useSubmitContact`

## 7. 后台前端模块

### `AuthContext`

- 保存 `auth_token` 到 `localStorage`
- 登录后拉取 `me`
- 统一提供：
  - `user`
  - `loading`
  - `isAuthenticated`
  - `login`
  - `logout`

### `AdminLayout`

- 负责后台鉴权守卫
- 负责桌面侧边栏与移动端抽屉菜单
- 当前菜单包括：概览、作品、视频、研学、分类、线索、媒体库、站点配置

### `DashboardPage`

- 使用 `Promise.allSettled` 并行拉取作品、视频、研学、联系、报名、媒体数量
- 支持部分失败：单个接口失败时该项计为 0，全部失败才报错
- 展示后台概览统计卡片

### `WorksManager`

- 作品列表
- 新增、编辑、删除
- 封面图通过 `ImageUploader` 管理
- 错误处理：Alert 通知

### `VideosManager`

- 视频列表
- 新增、编辑、删除
- 支持封面上传
- 错误处理：Alert 通知

### `WorkshopsManager`

- 活动列表
- 新增（跳转编辑器）、编辑（跳转编辑器）、删除
- 状态以前端映射方式展示
- 错误处理：Alert 通知

### `WorkshopEditorPage`

- 研学新建/编辑表单页
- 路由：`/admin/workshops/new` 与 `/admin/workshops/edit/:id`
- 富表单：活动信息、亮点、行程、费用等字段
- 封面上传复用 `ImageUploader`

### `CategoriesPage`

- 画廊分类管理
- 新增、编辑、删除分类
- 接通 API，非静态假数据

### `LeadsPage`

- 线索管理
- 查看联系表单提交与研学报名记录
- 支持状态筛选与详情查看

### `SiteConfigPage`

- 站点配置管理
- 编辑站点标题、简介、联系方式、社交链接等
- 保存后前台立即生效（Query 失效）

### `MediaPage`

- 上传图片
- 浏览媒体库（分页）
- 复制 URL
- 删除媒体记录

## 8. 数据流设计

### 公开内容

```text
组件 -> hooks -> api -> axios client -> mock 或真实 API
```

说明：

- `main.tsx` 已注入 `QueryClientProvider`（staleTime 5min）
- 根目录 `.env` 控制 `mock / real API`
- `api/client.ts` 中会统一处理 token 和响应解包

### 管理后台

当前管理后台多数页面直接使用 `apiClient` 请求，特点是：

- 实现简单
- 已够用
- 后续可逐步迁移到 Query + form schema 的更完整模式

## 11. 前端开发约束

### 动画约束

- 不要随意删除 `useGSAP`
- 不要随意替换 `Lenis`
- GSAP 插件统一在 `src/utils/gsap.ts` 注册
- `GallerySection` 改动后必须手动验证横向滚动
- 异步数据加载后必须调用 `ScrollTrigger.refresh()`

### 路由约束

- 旧链接 `/photographystudy/kenya-expedition` 仍需兼容
- 首页 hash 跳转依赖 `ScrollToTop`，不要随意简化
- 404 路由 `*` 必须放在最后

### 数据约束

- 公开内容优先复用 `hooks`
- 后端字段变更必须同步更新 `types/api.ts`
- 若新增接口，先更新 `docs/04-API接口文档.md`

### 上传约束

- 上传时不要手动设置 `Content-Type: multipart/form-data`，让 Axios 自动处理
- 上传组件统一复用 `ImageUploader`
- 大图上传前使用 `src/utils/imageCompress.ts` 客户端压缩

## 12. 当前已知前端不足

- Gallery 列表页仍为等高网格，未改为瀑布流
- 各 Section 入场动画统一用 `y:50 opacity:0`，缺乏节奏差异化
- 后台部分页面仍直接用 `apiClient`，未统一 Query 化
