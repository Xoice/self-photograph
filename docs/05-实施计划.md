## 后续实施计划

### 阶段 A：内容管理增强

**目标**：完善后台编辑能力，让管理员可以完整管理所有内容字段。

#### A1. 后台表单完善

- 作品表单：增加分类选择器、标签管理、描述编辑器、多图上传
- 视频表单：增加平台选择、时长校验、分类下拉
- 研学表单：增加行程编辑器、费用项编辑器、亮点编辑器
- 站点配置页：增加可视化编辑界面

#### A2. 分类与标签管理

- 后台分类管理页（CRUD + 排序 + 层级）
- 后台标签管理页（CRUD）
- 画廊列表页分类筛选功能化
- Navbar 画廊下拉菜单接真实分类数据

#### A3. 线索管理页

- 联系线索列表（分页、状态筛选、关键词搜索）
- 报名记录列表（分页、按活动筛选）
- 状态变更操作
- 基础统计

### 阶段 B：部署与生产化

**目标**：让项目可以安全部署到生产环境。

#### B1. 数据库迁移

- Prisma schema 切换 `provider` 为 `postgresql`
- 更新 `DATABASE_URL` 连接字符串
- 验证迁移兼容性

#### B2. 媒体存储迁移

- 接入 Cloudinary / S3 / OSS
- 保留本地上传作为 fallback
- 媒体删除同步清理云端文件

#### B3. 部署配置

- 前端构建 + Nginx 反向代理
- 后端 PM2 / Docker 部署
- HTTPS 配置
- 环境变量管理

### 阶段 C：SEO 与性能

**目标**：提升搜索引擎可见性和用户体验。

#### C1. SEO 增强

- 动态 meta 标签（react-helmet 或手动注入）
- sitemap.xml 生成
- robots.txt
- 结构化数据（JSON-LD）

#### C2. 性能优化

- 图片懒加载 + WebP 转换
- 代码分割优化（manualChunks）
- 首屏渲染优化
- Lighthouse 评分目标

### 阶段 D：质量保障

**目标**：建立测试和自动化体系。

#### D1. 测试框架

- Vitest 单元测试
- Playwright E2E 测试
- 关键路径覆盖

#### D2. CI/CD

- GitHub Actions
- 前端构建 + 后端构建 + 测试
- 自动部署

## 实施优先级

建议按 A1 → A2 → A3 → B1 → B2 → B3 → C1 → C2 → D1 → D2 顺序推进。

其中 A1（表单完善）是当前最影响日常使用的短板，应优先处理。
