# 前端页面修改建议

## 一、现状评估

### 整体判断

技术完成度很高（React 19 + MUI 7 + GSAP + Lenis + R3F），工程素质超出"个人站"水准。但从**摄影师个人网站**的设计视角看，当前视觉语言更像创意工作室/科技产品，与"摄影师展示作品"的核心叙事有偏差。

### 做得好的地方

- 暗色基底 #050505 让照片跳出画面，方向正确
- 画廊横向 pin 滚动是摄影站经典且有表现力的手法
- Lenis + GSAP 动效链路完整，滚动节奏有呼吸感
- Space Grotesk 字体有辨识度
- Three.js 粒子加位移场是一个有记忆点的 Hero
- ResponsiveImage 组件已预留 srcset 接口

### 核心问题（共 7 个，按修改优先级排列）

---

## 问题 1：荧光绿 #CCFF00 用得太满，和照片抢注意力

### 现状

#CCFF00 出现在：按钮文字/边框、链接 hover、Chip 背景、focus 轮廓、粒子颜色、导航下划线、研学卡片边框、分享按钮、Gallery 分类选中态、联系表单提交按钮。全站最亮的颜色是 UI 元素而不是照片。

### 问题

- 荧光绿太"科技感"，让人联想到赛博朋克/电竞品牌，和"城市与星空交汇"的摄影叙事气质冲突
- 在暗色背景上荧光绿过亮，持续吸引视觉焦点，分散对照片的注意力
一个摄影师的网站，最亮的颜色应该是照片本身

### 修改建议

- 将主色调从荧光绿 #CCFF00 改为更克制的暖琥珀色 #E0A458，或低饱和度的灰金 #C4A86A
- 暖色调与暗色基底搭配更有胶片质感，符合摄影主题
- 保留荧光绿仅用于极少量高优先级 CTA（如联系表单提交按钮），或者完全弃用
- 涉及文件：src/theme/index.ts 为主，各组件中硬编码的 #CCFF00 / primary.main 引用会自动跟随

---

## 问题 2：Hero 区更像产品发布会，不像摄影师的自我介绍

### 现状

12vw 超大标题 + fontWeight 900 + 全大写 + mixBlendMode exclusion 叠在粒子上。视觉冲击力强但气质偏硬。粒子效果是主视觉，没有真实照片。

### 问题

- 摄影师个人站的第一眼应该传递"这是一个摄影师"的信号
纯粒子+巨字的组合更像科技产品或创意机构
缺少真实摄影作品作为第一印象

### 修改建议

- 在粒子层之上叠加一张摄影师代表作作为全屏背景（低透明度渐变遮罩，让粒子在照片上微弱可见）
- 标题字号从 12vw 降到 8vw 左右，字重从 900 降到 700
- 副标题保留，但颜色从 primary.main 改为 text.primary 带轻微透明
- 粒子透明度降低，作为背景氛围而非主角
- 涉及文件：src/pages/Home/sections/HeroSection.tsx、src/components/three/HeroParticles.tsx

---

## 问题 3：Bio 区是纯文字，缺少摄影师人像

### 现状

Bio 区只有标题+段落+按钮，minHeight 100vh 但全是大段文字，没有照片。视觉上像一个 README 页面。

### 问题

"个人"站没有"人"，缺少个人感
纯文字区块在六个区块中缺乏视觉记忆点
摄影师的人像是建立信任和连接的关键

### 修改建议

- 改为左右分栏：左侧人像照片（竖构图），右侧文字
- 移动端改为上下布局：照片在上，文字在下
- 照片来源可以是站点配置里的一个新字段，或复用 bioImage 字段
- 照片可以带轻微的 GSAP 视差效果
- 涉及文件：src/pages/Home/sections/BioSection.tsx、src/types/api.ts（新增字段）、后端 site 模块

---

## 问题 4：Gallery 列表页是等高网格，没有发挥摄影作品的表现力

### 现状

GalleryListPage 用 Grid 等高网格布局，每张图都是固定 md:4（三列）或 lg:3（四列），高度统一。

### 问题

- 等高网格让所有作品看起来一样，失去了大小对比的叙事节奏
不同比例的摄影作品（横构图/竖构图/全景）被强制裁切到统一比例
缺乏编辑式排版的层次感

### 修改建议

- 改为 CSS columns 瀑布流布局，让图片保持原始比例
或者做编辑式混排：第一张大图占两列，后续小图占一列，交替排列
卡片悬停效果从 translateY(-8px) 改为更克制的纯透明度变化，减少 UI 噪音
- 涉及文件：src/pages/Gallery/GalleryListPage.tsx

---

## 问题 5：作品详情页图片展示偏"文档"而非"画廊"

### 现状

主图被 maxHeight 70vh 限制 + 1px solid rgba(234,234,234,0.1) 边框，看起来像文章插图。"作品介绍"标题用 color primary.main，注意力被拉到 UI 而非作品。

### 问题

- 高端摄影站通常用全宽或大幅面无边框图片，让作品自己说话
边框让图片看起来像被框住的内容而非展示品
maxHeight 限制让大图无法发挥视觉冲击力

### 修改建议

- 主图改为全宽无边框，去掉 maxHeight 限制，用 width 100% + 自然高度
- 点击放大保留，但去掉边框
- "作品介绍"标题颜色从 primary.main 改为 text.primary
-"更多图片"也改为全宽无边框
- 涉及文件：src/pages/Gallery/WorkDetailPage.tsx

---

## 问题 6：六个区块视觉节奏太均匀

### 现状

Bio、Gallery、Video、PhotoStudy、Contact 每个都是 minHeight 100vh + 居中标题 + 卡片网格，动画全是 y:50 opacity:0。滚下去感觉像翻同一页。

### 问题

- 缺乏"满屏照片 -> 留白文字 -> 紧凑网格 -> 大留白"的起伏节奏
所有区块动画完全一致，缺乏惊喜

### 修改建议

- Bio 区改为人像+文字分栏后，节奏自然变化
- Video 区可以改为更紧凑的列表式布局而非卡片网格
- Contact 区可以去掉卡片容器，改为更简洁的极简布局
- 动画做差异化：Bio 用横向滑入，Gallery 用缩放淡入，Contact 用逐字浮现
- 涉及文件：各 section 组件

---

## 问题 7：Footer 过于简单

### 现状

只有版权和"Designed with ♥ by React 19"。

### 问题

- 摄影师个人站 Footer 是放社交媒体、快速导航、联系方式的好位置
当前浪费了这个区域

### 修改建议

- 左侧：版权 + 品牌名
中间：快速导航（画廊、影视、研学、联系）
右侧：社交媒体图标（B站、微博等）
- 涉及文件：src/App.tsx Footer 组件

---

## 二、修改执行顺序

1. 问题 1 - 降荧光绿用量（影响最大、改动最小，改 theme 即可）
2. 问题 5 - 作品详情页主图全宽无边框（改动小、效果明显）
3. 问题 2 - Hero 加真实代表作背景（质变）
4. 问题 4 - Gallery 列表改瀑布流（让作品比例自由呼吸）
5. 问题 3 - Bio 区加人像照片（让"个人"站有人）
6. 问题 7 - Footer 增强
7. 问题 6 - 区块节奏差异化（改动最大，长期优化）
