### S1. [x]  [CRITICAL] 文件上传存储型 XSS — 双扩展名绕过

**漏洞描述：**
Multer 使用 `extname(file.originalname)` 提取扩展名，攻击者可上传名为 `payload.jpg.html` 的文件，将 Content-Type 设为 `image/jpeg`（在白名单中），Multer 接受上传，文件以 `.html` 扩展名保存。Express 静态服务以 `Content-Type: text/html` 提供该文件，浏览器执行其中的 JavaScript。

**已验证攻击路径：**
1. 攻击者上传 `image.jpg.html`（内容为 `<!DOCTYPE html><html><script>alert(1)</script></html>`），MIME 声明为 `image/jpeg`
2. 服务器返回 `{"code":0,"data":{"url":"/uploads/<uuid>.html",...}}`
3. 访问 `http://localhost:3000/uploads/<uuid>.html` → 200 OK，`Content-Type: text/html`
4. 浏览器执行 `<script>alert(1)</script>`

**影响：** 攻击者可在 API 域名下执行任意 JavaScript，窃取管理员 JWT Token（从 localStorage），发起任意 admin API 请求。

**涉及文件：** `apps/api/src/modules/media/media.module.ts:22-24`

```ts
// 当前代码 — 使用 originalname 的扩展名
filename: (_req, file, cb) => {
  cb(null, randomUUID() + extname(file.originalname));  // 漏洞：originalname 可被攻击者控制
},
```

**修复方案：**
```ts
const ALLOWED_EXTENSIONS: Record<string, string> = {
  'image/jpeg': '.jpg',
  'image/png': '.png',
  'image/gif': '.gif',
  'image/webp': '.webp',
};
// ...
filename: (_req, file, cb) => {
  const ext = ALLOWED_EXTENSIONS[file.mimetype] || '.bin';
  cb(null, randomUUID() + ext);
},
```

### S2. [x]  [HIGH] MIME 类型欺骗 — 不校验文件实际内容

**漏洞描述：**
Multer 的 fileFilter 仅检查 `file.mimetype`（来自 HTTP 请求头），不验证文件实际内容（magic bytes）。攻击者可上传 SVG（含 JS）、HTML、EXE 等文件，只需声明 MIME 为 `image/jpeg` 即可通过过滤。

**已验证：**
- SVG 内容（含 `<script>`）声明 `image/jpeg` → 上传成功 (201)
- HTML 内容声明 `image/png` → 上传成功 (201)
- EXE 内容声明 `image/png` → 上传成功 (201)

虽然这些文件以图片扩展名保存（`.jpg`/`.png`），浏览器不会执行其中的脚本（因为 `Content-Type: image/jpeg` + `nosniff`），但恶意文件仍被存储在服务器上。

**涉及文件：** `apps/api/src/modules/media/media.module.ts:26-31`、`apps/api/src/modules/media/media.service.ts`

**修复方案：** 在 MediaService 中添加 magic bytes 校验（已有 `parseImageDimensions` 读取文件头，可在此处增加内容类型验证），不匹配时删除文件并返回 400。

### S3. [x]  [HIGH] CSP 被禁用

**漏洞描述：**
Helmet 配置中 `contentSecurityPolicy: false`，未设置 Content-Security-Policy 头。CSP 是 XSS 的纵深防御层，在 React 转义失败时仍可阻止脚本执行。

**涉及文件：** `apps/api/src/main.ts:31-34`

```ts
app.use(helmet({
  contentSecurityPolicy: false,  // 应配置合适的 CSP 策略
  crossOriginEmbedderPolicy: false,
}));
```

**修复方案：**
```ts
app.use(helmet({
  contentSecurityPolicy: {
    directives: {
      defaultSrc: ["'self'"],
      scriptSrc: ["'self'"],
      styleSrc: ["'self'", "'unsafe-inline'"],
      imgSrc: ["'self'", 'data:', 'https:'],
      connectSrc: ["'self'", process.env.CORS_ORIGIN || 'http://localhost:5173'],
    },
  },
  crossOriginEmbedderPolicy: false,
}));
```
